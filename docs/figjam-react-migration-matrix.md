# FigJam React Cutover Migration Matrix

Issue: [#628](https://github.com/developer-1px/canvas/issues/628)

Architecture decision: [ADR 0005](./adr/0005-canonical-dom-react-editor-runtime.md)

## Status meaning

Every `[x]` row records the behavior actually shipped for the #628 internal
release candidate. It means the exposure resolves to a registered React widget
or structural definition, uses the canonical `DesignDocument`, and has the
listed regression evidence. It does not mean full legacy-product parity.

The 21 rows are product exposures and insertion aliases, not 21 distinct
renderer definitions. They converge on twelve widget definitions and the
`board`, `section`, and `group` structural definitions. Selection, viewport,
active tool, previews, and overlays remain ephemeral; authored nodes and their
stable references are persisted.

## Compatibility-only surface

- Public `CanvasItem`, `RectItem`, and `CanvasCustomItem` exports remain for the
  pre-1.0 compatibility surface.
- The default `/figjam` route does not read, render, or persist a parallel
  `CanvasItem` document. `/engine` remains a separate compatibility route.
- Custom item creation is not installed on `/figjam`, so `custom` is not one
  of the checked product exposures.

## In-document clipboard and reference policy

Selection copy/cut stores a canonical subtree in the current `/figjam` app
instance and places only an identifying marker on the system clipboard. Paste
and Duplicate consume that in-memory subtree in the same document; #628 does
not define selection transfer between documents, tabs, or app instances.

- Connector and comment references target stable design-node ids.
- If a referenced target is copied in the same selected set, the cloned
  reference points to the cloned target.
- If the target remains outside that set, the clone keeps the existing target
  id because both nodes are still in the same document.
- Reparenting preserves identity. Moving or resizing a target recomputes its
  attached presentation from the canonical read model.
- Deleting a referenced target materializes the last fallback geometry and
  clears that reference in the same reversible command. Undo and redo restore
  the corresponding state.
- HTML table, delimited text, URL, and ordinary text paste/drop are import
  paths, not serialized selection transport.

## Release-candidate evidence gate

The matrix is accepted for #628 only when:

1. `figjam-react-cutover.e2e.ts` covers default creation, direct and Shift
   selection, marquee, move/resize, text editing, drawing, connector creation,
   same-document clipboard, keyboard operations, undo/redo, pan/zoom, hidden
   palette exposures, persistence, and route isolation on `/figjam`.
2. `FigJamProductPack.test.tsx`, `FigJamDocumentOperations.test.ts`, and
   `FigJamAllFamiliesPersistence.test.ts` cover registered definitions,
   serializable geometry, stable connector/comment references, move, resize,
   reparent, delete, undo, redo, alias mapping, and restoration.
3. `CanvasFigJamCutoverBoundaries.test.ts` proves that `/figjam` uses the React
   product runtime without legacy route imports or a parallel `CanvasItem`
   document, keeps FigJam ids out of generic editor layers, and retains the
   public pre-1.0 compatibility export.
4. The existing FigJam interface remains content-first and reuses current
   styling with minimal borders and decoration.
5. Unit and architecture tests, lint, type checking, build, package smoke, and
   the full browser suite pass as the internal release-candidate gate.
6. No version bump, git tag, package publication, multiplayer backend, or
   marketplace work is included.
