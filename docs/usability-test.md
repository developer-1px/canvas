# 사용성 테스트 프로토콜

목적: 최근 개선이 실제 사용자에게 기능 발견과 작업 완료를 쉽게 만드는지 확인한다.

대상: 캔버스/문서 편집 도구를 써본 적이 있는 사용자 5명.

환경:

- URL: `http://127.0.0.1:5173/`
- 화면: 데스크톱 1280x800 우선, 필요 시 360x720 추가 확인
- 진행자 개입: 막히면 바로 알려주지 않고 중립 질문만 한다.

## 측정

각 과제마다 아래 값을 기록한다.

- 성공: `1` 성공, `0.5` 도움 후 성공, `0` 실패
- 시간: 과제 시작부터 완료까지 초 단위, 최대 180초
- 도움: 진행자가 힌트를 준 횟수
- 막힘: 5초 이상 멈춘 횟수
- 오조작: 의도와 다른 클릭/드래그/키 입력 횟수
- 난이도: 과제 후 사용자 점수, 1 쉬움 - 5 어려움

## 과제

1. 선택된 두 항목을 복제해서 오른쪽 빈 공간으로 옮긴다.
2. 방금 만든 복제 항목을 삭제한다.
3. 새 사각형을 만들고, 사각형 안의 글자를 `Hello`로 바꾼다.
4. 파란 사각형과 텍스트를 그룹으로 묶은 뒤 다시 해제한다.
5. 선택 항목에 화면을 맞춘 뒤 줌을 100%로 되돌린다.

## 진행 스크립트

시작:

> 지금부터 캔버스 편집 과제 5개를 해봅니다. 사용법 설명은 하지 않겠습니다. 생각나는 대로 말하면서 진행해 주세요.

막힘 질문:

> 지금 무엇을 찾고 있나요?

과제 후 질문:

> 어디에서 망설였나요?

> 예상과 달랐던 동작이 있었나요?

> 버튼 의미가 헷갈린 곳이 있었나요?

## 통과 기준

- 각 과제 성공률 80% 이상
- 각 과제 중앙값 60초 이하
- 사용자당 도움 요청 평균 1회 이하
- 난이도 평균 3 이하

기준 미달이면 해당 과제의 막힘 메모를 기준으로 다음 개선 항목을 만든다.
